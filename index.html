Overview

The goal of Architekt is to create a complete Project Management Software specifically tailored to the industrial construction industry. The intent is to avoid confusion in the distribution of information between the contractor and their subcontractor(s). Overall this increases profitability by decreasing time and labor and allows a project to run smoothly and more efficiently.

For the purpose of this presentation, we are assuming the term “contractor” is the party managing the overall project, and the term “subcontractor” is the party in control of all softcraft.

APP CONTENT OVERVIEW

1) Takeoff and Estimating : Isometrics are supplied to the contractor and scanned into the Architekt app. Each Isometric drawing is a line item in the app’s Main Database. The app uses image recognition to perform a material takeoff for each Isometric drawing to give a total cost estimate, total linear footage, and total material quantities.

2) Single Database Per Project : Once the database is created, the contractor is in control of the Main Database and is responsible for assigning each subcontractor’s scope of work and which information is available to them. The subcontractor can only view and update what the contractor has allowed. If information is updated by the subcontractor, the update will also be reflected in the Main Database and redistributed.

3) Document Library : Contains project files (pictures, markups, contracts, RFI’s, etc), that can be linked to each line item in the Main Database.

4) Project Timeline & Schedule : Creates a schedule(skyline) to notify subcontractors of their due dates.

FEATURE DETAILS

Takeoff and Estimating Detail Architekt scans and uploads Isometric drawings Architekt uses text recognition to extract the drawing number of each Isometric Drawing to create a line item in the Main Database. Architekt searches the Bill of Materials of each drawing for key words to get linear feet, craft specific specs, and material quantities. The extracted information is added to it’s corresponding line item in the Main Database. Architekt uses this information to calculate a cost estimate and a totals summary that can be sorted by craft. Subcontractor’s can use this information to determine job bid and budget.

Single Database Distribution : Now that the Main Database has been created, the contractor can begin refining the Main Database. The contractor assigns the drawings to Test Packages just as they traditionally would. Architekt allows the ability to attach files directly to the line item in the Main Database, such as pictures, RFIs, Isometrics, Revisions, Weld Mapping, etc.The contractor assigns the necessary columns of the Main Database to each subcontractor’s craft(s).

CONTINUED

This creates an individual database for each subcontractor department. Now each craft is using, viewing, and updating the information from the same source in their own personal database. When a contractor updates the Main Database, the information is passed “down” and updated in each subcontractor’s database. The idea is to keep everyone updated with the latest information. Similarly, when a subcontractor updates information in their own personal database, the updated information is passed back “up” to the Main Database for contractor approval. If the contractor approves the updated information, the new version will be passed down and updated in each subcontractor’s personal database.

Each subcontractor craft contains departments such as Production, Quality Control, Cost Control, etc. Therefore, a personal database for each craft is necessary.Again, all information and personal databases come directly from the contractor’s Main Database, and the data is constantly sent, received, and redistributed to all parties.
